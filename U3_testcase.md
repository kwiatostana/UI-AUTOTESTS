### **TC_LOGIN_UNI_001: Универсальная проверка авторизации (параметризованный)**

**Цель:** Проверить корректность работы механизма авторизации при использовании различных наборов учетных данных.

**Предусловие:**
1. Открыт браузер.
2. Осуществлен переход на страницу авторизации: `https://www.way2automation.com/angularjs-protractor/registeration/#/login`.

**Тестовые данные (Parameters):**

- Успешный вход: Логин angular, Пароль password. Ожидаем успех.
- Неверный пароль: Логин angular, Пароль invalid_password. Ожидаем ошибку.
- Неверный логин: Логин invalid_user, Пароль password. Ожидаем ошибку.

*(Примечание: Поле `Username Description`, обязательное на этой форме, заполняется значением из поля `Username`)*

**Шаги:**
1. Ввести значение **Username** из текущего *Набора* в поле "Username".
2. Ввести значение **Password** из текущего *Набора* в поле "Password".
3. Ввести значение **Username** в поле "Username Description".
4. Нажать кнопку **Login**.

**Ожидаемые результаты:**

*   **Если Ожидаемый результат (Тип) == Success:**
    *   Появляется сообщение об успехе.
    *   Текст сообщения содержит **Ожидаемое сообщение** (из таблицы).
    *   Пользователь перенаправлен на защищенную страницу (или URL изменился).

*   **Если Ожидаемый результат (Тип) == Error:**
    *   Появляется сообщение об ошибке (alert).
    *   Текст сообщения полностью совпадает с **Ожидаемое сообщение** (из таблицы).
    *   Пользователь остается на странице логина.

**Постусловие:**
*   Если вход был успешным (Set 1): Выполнить **Logout**, чтобы вернуть приложение в исходное состояние для следующих тестов.
